# Design: admin-console → demo-hub 产品配置集成

Generated by /brainstorming on 2026-05-15
Branch: main
Status: APPROVED

---

## 功能概述

admin-console 通过 Supabase 的 `demo_hub_products` 表管理 demo-hub 首页的产品展示配置。开发者负责编写集成代码（路由 + 视图），admin-console 负责控制哪些产品可见、名称叫什么、排列顺序如何。

---

## 核心原则

- **代码层与展示层分离**：`product_key`（路由 slug）绑定代码，admin 只读；`display_name`、`description`、`enabled`、`sort_order` 由 admin 控制
- **配置通过 Supabase 传递**：admin-console 写，demo-hub 读，两个服务不直接通信
- **启动时加载，重启生效**：demo-hub 启动时一次性从 Supabase 拉取配置到内存，简单可靠

## 数据库设计

### 表：`demo_hub_products`

```sql
CREATE TABLE demo_hub_products (
  id           uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  provider     text        NOT NULL CHECK (provider IN ('paypal','braintree','stripe','adyen')),
  sdk_version  text        NOT NULL,  -- 'jssdk-v6' | 'jssdk-v5' | 'web-sdk' | 'graphql' 等
  product_key  text        NOT NULL,  -- 叶子产品 slug，如 'acdc' | 'paypal-button'
  display_name text        NOT NULL,  -- 首页卡片和产品页 <title> 显示名称
  description  text        NOT NULL DEFAULT '',  -- 首页卡片一句话描述
  enabled      boolean     NOT NULL DEFAULT true,
  sort_order   integer     NOT NULL DEFAULT 0,
  created_at   timestamptz NOT NULL DEFAULT now(),
  updated_at   timestamptz NOT NULL DEFAULT now(),

  UNIQUE(provider, sdk_version, product_key)  -- 三字段联合唯一
);

-- 自动更新 updated_at
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN NEW.updated_at = now(); RETURN NEW; END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER demo_hub_products_updated_at
  BEFORE UPDATE ON demo_hub_products
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

## demo-hub 端实现

### 启动时配置加载（`src/config/products.js`）

```js
// 产品配置内存缓存，启动时填充，运行时只读
let productConfig = new Map()

async function loadProductConfig(supabase) {
  const { data, error } = await supabase
    .from('demo_hub_products')
    .select('*')
    .order('provider')
    .order('sort_order')

  if (error) throw new Error(`Failed to load product config: ${error.message}`)

  productConfig = new Map(
    data.map(row => [`${row.provider}/${row.product_key}`, row])
  )
}

function getProduct(provider, productKey) {
  return productConfig.get(`${provider}/${productKey}`) ?? null
}

function getEnabledProducts() {
  return [...productConfig.values()].filter(p => p.enabled)
}

module.exports = { loadProductConfig, getProduct, getEnabledProducts }
```

### app.js 启动顺序

```js
const { loadProductConfig } = require('./config/products')
const { createClient } = require('@supabase/supabase-js')

const supabase = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_SERVICE_KEY)

async function start() {
  await loadProductConfig(supabase)  // 启动时加载，失败则进程退出
  app.listen(process.env.PORT || 3000)
}
start()
```

### 首页路由（`routes/index.js`）

```js
const { getEnabledProducts } = require('../config/products')

router.get('/', (req, res) => {
  const products = getEnabledProducts()
  // 按 provider 分组
  const grouped = products.reduce((acc, p) => {
    if (!acc[p.provider]) acc[p.provider] = []
    acc[p.provider].push(p)
    return acc
  }, {})
  res.render('index', { grouped })
})
```

### 产品页标题（各路由文件示例）

```js
const { getProduct } = require('../../config/products')

router.get('/paypal/acdc', (req, res) => {
  const config = getProduct('paypal', 'acdc')
  const title = config?.display_name ?? 'PayPal ACDC'
  res.render('paypal/acdc', { title, amount: '1.00' })
})
```

## admin-console 端实现

### API 路由（Express 后端）

```
GET  /api/demo-hub/products          → 列出所有产品（含 disabled）
PATCH /api/demo-hub/products/:id     → 更新 display_name / description / enabled / sort_order
POST  /api/demo-hub/products/:id/move → 调整排序（up / down）
```

**不提供 POST（创建）和 DELETE（删除）接口** — 产品的增删由开发者直接操作数据库，admin 只管展示配置。

## 边界 & 安全

| 场景 | 处理方式 |
|------|---------|
| Supabase 中有产品但 demo-hub 无对应路由 | 首页不渲染该产品（safe fallback） |
| 启动时 Supabase 不可达 | 进程报错退出，避免以空配置运行 |
| admin-console 尝试修改 product_key | 后端拒绝，返回 400 |
| admin 未登录访问管理接口 | Supabase Auth 拦截，返回 401 |

---

## 变更生效流程

```
admin-console 修改配置
  → PATCH /api/demo-hub/products/:id
  → 写入 Supabase demo_hub_products
  → 运维/开发者重启 demo-hub
  → demo-hub 启动，重新从 Supabase 加载配置
  → 首页反映最新配置
```
