## 背景与问题

原始设计将 `jssdk-v5` / `jssdk-v6` 视为叶子路由节点，但它们实际上是 **SDK 版本大类**，每个大类下有多个独立的支付产品（PayPal Button、PayLater、ACDC 等），且同一产品在不同 SDK 版本下的集成方式（初始化、API、语法）差异显著，需要独立 demo 页面。

所有 provider（PayPal、Braintree、Stripe、Adyen）均预留三层路由，未来可灵活扩展。

## 路由设计：三层结构（全部 provider 统一）

```
/{provider}/{sdk_version}/{product_key}
```

### PayPal

```
GET /paypal/jssdk-v6/paypal-button    → PayPal Button (JSSDK v6)
GET /paypal/jssdk-v6/paylater         → PayLater Button
GET /paypal/jssdk-v6/venmo            → Venmo
GET /paypal/jssdk-v6/bcdc             → Basic Credit/Debit Card
GET /paypal/jssdk-v6/acdc             → Advanced Credit/Debit Card
GET /paypal/jssdk-v6/apple-pay        → Apple Pay
GET /paypal/jssdk-v6/google-pay       → Google Pay
GET /paypal/jssdk-v6/vault            → Vault
GET /paypal/jssdk-v5/paypal-button    → PayPal Button (JSSDK v5)
GET /paypal/jssdk-v5/paylater         → PayLater (v5)
GET /paypal/jssdk-v5/acdc             → ACDC (v5)
... (更多 v5 产品待补充)
```

### Braintree

```
GET /braintree/web-sdk/dropin-ui       → Braintree Drop-in UI
GET /braintree/web-sdk/hosted-fields   → Hosted Fields
GET /braintree/graphql/...             → GraphQL API 集成（待扩展）
```

### Stripe（预留）

```
GET /stripe/stripe-js/payment-element  → Stripe Payment Element
GET /stripe/stripe-js/card-element     → Card Element
... (待定)
```

### Adyen（预留）

```
GET /adyen/web-components/card         → Adyen Card Component

```

## Supabase `demo_hub_products` 表更新

### Schema 变更

```sql

ALTER TABLE demo_hub_products
  ADD COLUMN sdk_version text NOT NULL DEFAULT '';

-- 重建唯一约束（三字段组合唯一）
ALTER TABLE demo_hub_products
  DROP CONSTRAINT demo_hub_products_provider_product_key_key;

ALTER TABLE demo_hub_products

    UNIQUE (provider, sdk_version, product_key);
```

## demo-hub 内存配置 Map 结构更新

```js
// 启动时构建：key = 'provider/sdk_version/product_key'
productConfig = new Map([
  ['paypal/jssdk-v6/paypal-button', { displayName, description, enabled, sortOrder }],
  ['paypal/jssdk-v6/acdc',          { ... }],
  ['braintree/web-sdk/dropin-ui',    { ... }],
  ...
])

// URL 生成辅助函数
function productUrl({ provider, sdk_version, product_key }) {
  return `/${provider}/${sdk_version}/${product_key}`
}

// 路由内查询配置
function getProduct(provider, sdkVersion, productKey) {
  return productConfig.get(`${provider}/${sdkVersion}/${productKey}`) ?? null
}
```

## admin-console 影响

`demo_hub_products` 表新增 `sdk_version` 字段，admin-console 产品列表页需要：
- 按 `provider → sdk_version → product_key` 三级分组展示
- `sdk_version` 对用户可见（展示为分组标题），但与 `product_key` 一样**不可编辑**
- 可编辑字段仍只有：`display_name`、`description`、`enabled`、`sort_order`

## 命名规范汇总

| 层级 | 命名规则 | 示例 |
|------|----------|------|
| provider | 小写，无连字符 | `paypal`, `braintree`, `stripe`, `adyen` |
| sdk_version | 小写 + kebab-case | `jssdk-v6`, `jssdk-v5`, `web-sdk`, `graphql`, `stripe-js`, `web-components` |
| product_key | 小写 + kebab-case | `paypal-button`, `acdc`, `apple-pay`, `dropin-ui` |
| 文件名 | = product_key | `acdc.js`, `apple-pay.js`, `dropin-ui.js` |
| URL | `/{provider}/{sdk_version}/{product_key}` | `/paypal/jssdk-v6/acdc` |
