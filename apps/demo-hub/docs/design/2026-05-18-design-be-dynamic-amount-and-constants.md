Generated by /brainstorming on 2026-05-18
Branch: main
Status: APPROVED

## 概述

两个相互关联的改进：

1. **动态金额输入** — 沙盒页面金额从硬编码改为可输入，默认 $100.00，买家可自定义，前端验证后传给后端
2. **API 常量文件** — 提取所有 PayPal API 调用中重复出现的基础参数值为命名常量，新增 `buildOrderBody()` 辅助函数统一组装 order body

### `buildOrderBody(amount, overrides)` 辅助函数

统一组装标准 PayPal order body，各路由文件调用此函数，不再手写 body：

```js
function buildOrderBody(amount, overrides = {}) {
  const value    = String(parseFloat(amount).toFixed(2))
  const currency = overrides.currency || DEFAULT_CURRENCY

  return {
    intent: INTENT.CAPTURE,
    purchase_units: [{
      amount: {
        currency_code: currency,
        value,
        breakdown: {
          // 如包含 items，PayPal 要求 breakdown.item_total 与 items 之和相等
          item_total: { currency_code: currency, value }
        }
      },
      description: DEMO_DESCRIPTION,
      items: [{
        ...DEMO_ITEM,
        unit_amount: { currency_code: currency, value },
      }],
      shipping: SANDBOX_SHIPPING,
      // 允许各产品覆盖 purchase_unit 内的字段（如 Vault payment_source）
      ...(overrides.purchaseUnit || {}),
    }],
    // 允许覆盖顶层字段（如 payment_source 在顶层的场景）
    ...(overrides.topLevel || {}),
  }
}

module.exports = {
  INTENT, CURRENCY, ITEM_CATEGORY,
  DEFAULT_AMOUNT, DEFAULT_CURRENCY,
  DEMO_DESCRIPTION, DEMO_ITEM,
  SANDBOX_SHIPPING, SANDBOX_BILLING,
  buildOrderBody,
}
```

### 前端验证规则

| 规则 | 说明 |
|------|------|
| 非空 | 不能为空 |
| 正数 | 必须 > 0 |
| 数字格式 | 只允许数字和一个小数点 |
| 最多两位小数 | 不允许 `100.123` |
| 最小值 | ≥ $0.01 |

验证时机：`blur`（失焦）时验证并格式化（自动补两位小数），SDK 按钮点击时二次验证。

### 前端传参（JS 文件改动）

各 `createOrder` callback 读取输入值传给后端：

```js
createOrder: function () {
  var amountInput = document.getElementById('demo-amount')
  var amount = amountInput ? amountInput.value.trim() : window.DEMO.defaultAmount

  // 格式化：确保两位小数
  amount = parseFloat(amount).toFixed(2)

  return fetch(urls.createOrder, {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify({ amount: amount }),
  })
  .then(function (r) { return r.json() })
  .then(function (d) {
    if (d.error) throw new Error(d.error)
    return d.id
  })
}
```

### 后端改动（`_factory.js`）

POST handler 读取 `req.body.amount`，调用 `buildOrderBody`：


## Part 3：各产品 create-order body 迁移

| 路由文件 | 当前做法 | 迁移后 |
|---------|---------|--------|
| `_factory.js` (createStandardRoute) | 硬编码或 `...orderBody` | `buildOrderBody(req.body.amount, { topLevel: orderBody })` |
| `_factory.js` (createVaultWithPurchaseRoute) | 硬编码含 `payment_source` | `buildOrderBody(req.body.amount, { purchaseUnit: { payment_source } })` |
| `buttons.js` | 手写 intent + purchase_units | `buildOrderBody(req.body.amount)` |
| `acdc.js` | 手写 intent + purchase_units | `buildOrderBody(req.body.amount)` |
| `vault-paypal-setup-only.js` | `/v3/vault/setup-tokens`（不需要 amount） | 不变 |
| `vault-acdc-setup-only.js` | `/v3/vault/setup-tokens` | 不变 |
| `vault-return.js` | 手写含 payment_source.token | `buildOrderBody(req.body.amount, { topLevel: { payment_source: {...} } })` |

## 数据流

```
用户在输入框填写金额
  │
  ▼ (blur) 前端验证 + 格式化 → "99.99"
  │
  ▼ 用户点击 PayPal 按钮 → createOrder callback
  │
  ▼ fetch POST { amount: "99.99" }
  │
  ▼ Express POST handler
    ├─ req.body.amount = "99.99"
    ├─ buildOrderBody("99.99")
    │   ├─ intent: INTENT.CAPTURE
    │   ├─ purchase_units[0].amount.value = "99.99"
    │   ├─ purchase_units[0].items[0].unit_amount.value = "99.99"
    │   ├─ purchase_units[0].description = DEMO_DESCRIPTION
    │   └─ purchase_units[0].shipping = SANDBOX_SHIPPING
    │
    └─ POST /v2/checkout/orders → PayPal API
```

---

## 不在此次范围内

- 多商品支持（items 数组多项）
- 税费、折扣的 breakdown 细项
- 动态货币切换（当前固定 USD）
- Billing address 字段展示（PayPal 从买家账户获取，ACDC 后续单独处理）
- 金额上限限制（sandbox 无上限）
