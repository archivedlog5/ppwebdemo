## 项目定位

demo-hub 是一个**支付产品集成演示中心**，以最简洁的方式展示各支付提供商（PayPal、Braintree、Stripe、Adyen）的不同支付产品如何集成。

**核心原则：越简单越好。** 每个 demo 页面就是支付 widget 本身，无商品卡、无购物车、无电商 UI。

## 核心需求

### 1. 首页（产品目录）

- 展示所有支持的支付产品，按提供商分组：PayPal → Braintree → Stripe → Adyen
- 每个产品卡显示：提供商名称、产品名称、简短描述（1 句话）、进入 demo 的链接
- 点击产品卡进入该产品的独立 demo 页面

### 2. 产品 Demo 页面（每个产品独立路由）

**页面内容极简：**
- 页面标题：产品名称（如 "PayPal ACDC"、"Braintree Drop-in UI"）
- 测试金额显示（如 "Pay $1.00 - Test"）
- 支付 widget：按钮 / 卡片输入框 / Apple Pay 按钮 / 等——产品本身决定
- 支付结果反馈（成功/失败消息）
- **无商品图片、无购物车、无导航栏电商元素**

**Tab 结构（架构预留，现阶段只激活 Demo Tab）：**
```
[ Demo ] [ Code ] [ Logs ]
          (禁用)   (禁用)
```
后续激活 Code Tab 无需改动每个产品页，只修改共享 layout。

### 3. 路由规范

```
GET /                            → 首页产品目录
GET /paypal/jssdk-v5             → PayPal JSSDK v5 demo
GET /paypal/jssdk-v6             → PayPal JSSDK v6 demo
GET /paypal/acdc                 → PayPal Advanced Credit/Debit Card demo
GET /paypal/applepay             → PayPal Apple Pay demo
GET /paypal/googlepay            → PayPal Google Pay demo
GET /paypal/vault                → PayPal Vault demo
GET /paypal/apm                  → PayPal APM demo
GET /paypal/invoice              → PayPal Invoice demo
GET /braintree/dropin-ui         → Braintree Drop-in UI demo
GET /braintree/hosted-fields     → Braintree Hosted Fields demo
GET /stripe/<product>            → Stripe 产品 demo（待定）
GET /adyen/<product>             → Adyen 产品 demo（待定）

POST /api/paypal/create-order    → 创建 PayPal 订单（后端）
POST /api/paypal/capture-order   → 捕获 PayPal 订单（后端）
POST /api/braintree/client-token → 生成 Braintree Client Token（后端）
... 以此类推，每个产品有对应的后端 API 路由
```

## 技术架构

### 选定方案：EJS Partials + 三层路由 + 工厂函数

```
apps/demo-hub/
├── src/
│   ├── app.js                   # Express 入口，挂载所有路由
│   ├── config/
│   │   ├── products.js          # 内存 Map（启动时从 Supabase 填充）
│   │   └── paypal.js            # Access token helper（CN+US，8h TTL 缓存）
│   ├── routes/
│   │   ├── index.js             # GET / → 首页
│   │   └── paypal/
│   │       └── jssdk-v5/
│   │           ├── _factory.js  # createStandardRoute + createVaultWithPurchaseRoute
│   │           ├── spb-ecm.js   # 工厂路由示例
│   │           └── ...          # 共 14 个产品路由
│   ├── views/
│   │   ├── partials/
│   │   │   ├── header.ejs       # HTML head + topbar + sidebar（含 Tab 栏预留）
│   │   │   ├── footer.ejs       # 关闭标签
│   │   │   └── sidebar.ejs      # 左侧产品列表
│   │   ├── index.ejs            # 首页产品目录
│   │   └── paypal/
│   │       └── jssdk-v5/        # 每产品一个 EJS 文件
│   └── public/
│       ├── css/
│       └── js/
├── .env
├── .env.example
├── package.json
└── docs/
```

### 关键设计原则

1. **一个路由文件 = 一个支付产品**，不允许跨产品共用路由逻辑
2. **凭证/密钥从环境变量读取**，通过 `.env` 管理，绝不 hardcode
3. **EJS 布局**：`<%- include('../partials/header') %>` + 内容 + `<%- include('../partials/footer') %>`（不使用单一 layout.ejs）
4. **工厂函数**：`createStandardRoute` / `createVaultWithPurchaseRoute` 统一 GET + POST 路由逻辑
5. **每个产品路由包含：** 页面渲染 + 该产品所需的后端 API

## 前提（已确认）

1. 凭证从环境变量 / Supabase 读取，不 hardcode
2. 每个产品路由同时处理页面渲染和该产品的后端 API 调用
3. Tab 结构（Demo | Code | Logs）在 layout 层预留，现在只激活 Demo
4. 首页按提供商分组展示所有产品，每个产品有独立可分享 URL
5. Demo 页面极简：只有支付 widget + 测试金额，无电商 UI 元素

## admin-console 配置集成（已确认）

admin-console 通过 Supabase `demohub.products` 表控制 demo-hub 的首页展示内容。

### 数据表结构

```sql
id           uuid        PRIMARY KEY DEFAULT gen_random_uuid()
provider     text        NOT NULL  -- 'paypal' | 'braintree' | 'stripe' | 'adyen'
product_key  text        NOT NULL  -- 路由 slug，如 'acdc'（只读，绑定代码路由）
display_name text        NOT NULL  -- 首页和产品页显示名称（admin 可改）
description  text                  -- 首页卡片描述（admin 可改）
enabled      boolean     NOT NULL DEFAULT true
sort_order   integer     NOT NULL DEFAULT 0
created_at   timestamptz NOT NULL DEFAULT now()
updated_at   timestamptz NOT NULL DEFAULT now()

UNIQUE(provider, product_key)
```

### 启动时配置加载流程

```
app.js 启动
  → 从 Supabase 查询所有产品配置（含 sdk_version 字段）
  → 存入内存 Map：
    key = 'paypal/jssdk-v5/spb-ecm'  (provider/sdk_version/product_key)
    value = { displayName, description, enabled, sortOrder, ... }
  → 首页渲染：只展示 enabled=true 的产品，按 provider → sdk_version → sortOrder 分组
  → 产品页标题：getProduct(provider, sdkVersion, productKey) → displayName
```

### 配置变更生效方式

admin-console 修改 → 写入 Supabase → **重启 demo-hub** → 生效

### 新增产品完整流程

1. 开发者写路由文件（`routes/paypal/new-product.js`）和视图（`views/paypal/new-product.ejs`）
2. 在 Supabase `demohub.products` 插入一行（`product_key` = 路由 slug）
3. 重启 demo-hub
4. admin-console 中可见并可配置该产品的展示内容
